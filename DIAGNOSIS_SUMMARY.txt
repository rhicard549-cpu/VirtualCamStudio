DEEP DIAGNOSIS RESULTS - ROOT CAUSE IDENTIFIED AND FIXED
=============================================================================

PROBLEM SUMMARY:
CloudPhone and webcamtests were showing incorrect colors (purple/green) instead 
of the actual uploaded media from VirtualCamStudio.

ROOT CAUSE FOUND:
A pixel format mismatch caused a DOUBLE COLOR SWAP bug in the pipeline.

THE BUG IN DETAIL:
==================

1. VirtualCamStudio was sending frames in BGRA format (Blue, Green, Red, Alpha)
2. But it marked them as format 0, which means RGBA according to FrameIPC.h
3. UnityCaptureSender received the data and believed it was RGBA
4. UnityCapture driver (UnityCaptureFilter.cpp line 226) then converted 
   "RGBA" to BGRA by swapping Red and Blue channels
5. RESULT: Red and Blue got swapped TWICE = wrong colors!

PROOF FROM CODE ANALYSIS:
=========================

File: Research/UnityCapture/Source/UnityCaptureFilter.cpp
Line 226: #define RGBATOBGRA(x) ((x&0xFF00FF00)|((x&0x00FF0000)>>16)|((x&0x000000FF)<<16))

This macro SWAPS the R and B channels because UnityCapture EXPECTS RGBA input
and converts it to BGRA for Windows display.

File: UnityCaptureSender/FrameIPC.h  
Line 20: uint32_t pixelFormat; // 0 = RGBA32

Format 0 is defined as RGBA, not BGRA.

File: VirtualCamStudio/Outputs/UnityCaptureOutput.cs (BEFORE FIX)
Line 154: BitConverter.GetBytes(0).CopyTo(headerBytes, 16); // PixelFormat: 0 = BGRA32
Line 156: // Get pixel data (already in BGRA format)

MISMATCH! Sending BGRA but marking as RGBA!

THE FIX APPLIED:
================

Modified: VirtualCamStudio/Outputs/UnityCaptureOutput.cs

BEFORE:
-------
- Sent BGRA data directly from bgraFrame.Data
- Marked as format 0 (which means RGBA)
- No conversion happening

AFTER:
------
- Added explicit BGRA to RGBA conversion using OpenCV:
  Mat rgbaFrame = new Mat();
  Cv2.CvtColor(bgraFrame, rgbaFrame, ColorConversionCodes.BGRA2RGBA);

- Now sends RGBA data from rgbaFrame.Data
- Still marked as format 0 (RGBA) - now CORRECT!
- Added proper cleanup: rgbaFrame.Dispose();

EXPECTED RESULT:
================

Pipeline flow NOW (CORRECT):
1. Studio: BGRA frame → convert to RGBA → send as format 0 ✓
2. Sender: receive RGBA (matches format 0) ✓
3. UnityCapture: convert RGBA to BGRA (one swap) ✓
4. CloudPhone: displays correct colors! ✓
